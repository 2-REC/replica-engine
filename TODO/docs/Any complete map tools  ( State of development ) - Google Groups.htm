levels for Replica Island.

*** Flash Level Editor ***

I've spotted the Flash Level Editor built by CJ , but could not get it
to work. I changed all the necessary paths, still couldn't get it to
work. Noticed that the thread was dated July 17th, 2010, which was
like 8 months ago. Is the Editor still under dev, and how could I get
it to work?

Link:
http://groups.google.com/group/replica-island-c4oding-community/browse_thread/thread/dd163179b1a097ed/c3a8b7fc1da4809f?lnk=gst&q=level#

written (Qt Exporter, etc) and a lot of pixie dust which (as a noob) I
don't understand. I do understand the Replica Island Java code , but
once it comes to maps, I'm kind of paralyzed. I think it has been a
MAJOR hindrance in people pursuing using the game engine to make
awesome games.

Link:
http://groups.google.com/group/replica-island-coding-community/browse_thread/thread/82f4ef239f6e0f1b/7c514e2f40657147?lnk=gst&q=flash+level+editor#7c514e2f40657147


*** Final ***

I do understand that Chris has a huge workload, what does the
community think? Are we getting a simple, easy to use GUI Level Editor
with collision layers? My background has never been in deep level hex/
binary stuff, I'm more of a creative person and really need a proper
level making tool. ( like Chris said in the forums,"level making" is
the meat of any game )

Perhaps someone should write up a "step by step" tutorial on how to
make maps the binary way, make it so simple that nothing can possibly
go wrong. That could be a start.

I am also looking at alternative game engines, and AndEngine comes to
mind. The community is pretty active, and the developer is coming up
with nice level editor (hasn't been released yet).

My suggestion is to use Tiled or something similar and get a plug-in  
that works.

Map editors take forever to write correctly, which is why so many of  
them (mine included) are totally half-assed.

Alternatively, you could write a new loader.  The RI map format is  
*extremely* simple.  There's no reason you couldn't fill out that  
information with a text file, or XML, or some other format that  
already exists.  It's pretty much just tile indexes with a few rules.

Creating and customizing levels for Replica Island

I have been able to successfully create a level from scratch using Tiled and the plug-in that is available on the RI coding community blog. This isn’t a full walkthrough, you will find information for the installs and setting up Eclipse, getting the source, etc. elsewhere.

These are just tips that I think will help answer the things that may stump you after you get set up. This is all just to jump start your understanding of how to create levels using the original image and configuration files. It’s just a way to modify what’s already there which can be fun but maybe it’s also enough to get you over the hurdle and see what’s possible for your own game.

The links:

Tiled homepage

 <http://www.mapeditor.org/>

But you don’t want the latest you want this link which takes you to version 0.5.0. This is the version which is compatible with the plug-in:

http://downloads.sourceforge.net/project/tiled/tiled-qt/0.5.0/tiled-qt-0.5.0-win32.zip

This is the plug-in for Tiled that allows it to export the RI format bin file.

RI-Tiled-Qt-Exporter.rar

http://www.google.com/url?sa=D&q=http://replica-island-coding-community.googlegroups.com/web/RI-Tiled-Qt-Exporter.rar

Thank you to the person who wrote and posted the Tiled plug-in to this discussion group.  I have only seen the link to the uploaded .rar file so I don’t know who wrote it.

The sample map that comes with the plug-in did not work for me, February 2011. It had bizarre scenery and crashed the app., Try my sample instead and if nothing has changed on the game since February 2011, it should work.
Binary File Format

Attached is a spreadsheet explaining a little about the format of the binary file and the values to use for the object and hotspot layers to get the results you want.

Using Tiled

Painting in Tiled is like using any other painting app. There are simple painting tools. The “palette” is in the lower right corner and is called Tileset. The canvas is called the map. The layers are shown on the upper right. Each layer has special attributes which you can access by right clicking on the layer and choosing “Layer Properties…”.

Tilesets

Tilesets from RI can be loaded into Tiled in order to paint with them.  Use Map; New Tileset… on the Tiled main menu to load in a png file from the res\drawable folder. There are images there which can be used as Tilesets. These are the images: collision.png, grass.png, island.png, sewage.png, cave.png, lab.png.  

In a raster image editing tool you can edit the png files or make your own, keeping to the same file dimensions and keep your images fitting correctly in a grid of 32 x 32 pixels per tile. You can then just overwrite the original file or reconfigure the game to recognize and use your file.

The Layers

Paint the collision layer with the collision tileset.

It doesn’t matter which tileset you paint the object or hotspot layer with, as long as you paint with the tile that has the correct index value. So it will look odd to see pieces of tree or scenery on those layers but they are not going to show in the game, they are just placeholders to provide the index value in the final binary file.

The tiles on the map start at index 0 in the upper left corner, count across and then down.  So if you are trying to create a spawn spot for a coin, which is map index 1, you would paint onto the map, with the object layer selected, whichever tile is the second from the upper left on the tileset you are using.

Order of Layers

Set up the layers in Tiled to this order, it helps with setting up the scenery layers and it seems to have some effect on layer order in the final binary file which is important:

Collision –sets the boundaries for character movement.
Foreground – where the characters interact. Should match the layout of the collision layer.
Midground – clouds, distant landscape, etc.
Object – set the spawn targets where objects and characters will appear.
Hotspot – active areas to affect the characters.

Background

The Background is specified in Tiled by setting a map variable called backgroundIndex.

Save your map

 File; Save As… Tiled Map Files. This is the format that you can come back and edit later.

Export as a Replica Island binary

File ; Export As… and choose Replica Island Map File

To test your created level

First make a copy of the first level of RI.

It is a .bin file with this filename: \res\raw\level_0_1_sewer.bin

You may have to make a copy outside of the folder it’s in otherwise Eclipse will give warning messages because it tries to interpret the file.

Then overwrite level_0_1_sewer.bin with the exported bin file from the Tiled application.

Rebuild in Eclipse and Run it on your android phone.

Thank you

Thank you Chris for making your source code available, very generous of you.

Thank you to all the folks who posted information on level editing to the Replica Island discussion group, it was very helpful.

Thank you Chris!
 
It's much nicer to work with images like that on the tileset when painting a map. Less chance to mistake an object for scenery.
 
I have updated the reference spreadsheet to include images in the list of hotspots and objects. Cleaned up some of the information too.
 
I have also uploaded two tileset image files that show the tile index value for each tile. One hex, one decimal.
I think this can be useful when painting placeholders for items that will be defined later.
 
I also just tested this little trick:
 Tiled keeps a reference to the image file specified when setting up a new tileset. So if that image file changes, it will show up as changed in Tiled.
 I overlaid the object image file with a hex numbered image file.
 I restart Tiled and now I see the map with the hex values of the tiles instead of the images.
 This may be useful if I am troubleshooting the app and want a quick way to know what values I should be seeing in the data.
 I copy the original image file back and the object images are displayed again when I restart Tiled.

> painting tools. The “palette” is in the lower right corner and is called
> Tileset. The canvas is called the map. The layers are shown on the upper
> right. Each layer has special attributes which you can access by right
> clicking on the layer and choosing “Layer Properties…”.
>
> *Tilesets*
>
> Tilesets from RI can be loaded into Tiled in order to paint with them.  Use
> Map; New Tileset… on the Tiled main menu to load in a png file from the
> res\drawable folder. There are images there which can be used as Tilesets.
> These are the images: collision.png, grass.png, island.png, sewage.png,
> cave.png, lab.png.  
>
> In a raster image editing tool you can edit the png files or make your own,
> keeping to the same file dimensions and keep your images fitting correctly
> in a grid of 32 x 32 pixels per tile. You can then just overwrite the
> original file or reconfigure the game to recognize and use your file.
>
> *The Layers*
>
> Paint the collision layer with the collision tileset.
>
> It doesn’t matter which tileset you paint the object or hotspot layer with,
> as long as you paint with the tile that has the correct index value. So it
> will look odd to see pieces of tree or scenery on those layers but they are
> not going to show in the game, they are just placeholders to provide the
> index value in the final binary file.
>
> The tiles on the map start at index 0 in the upper left corner, count across
> and then down.  So if you are trying to create a spawn spot for a coin,
> which is map index 1, you would paint onto the map, with the object layer
> selected, whichever tile is the second from the upper left on the tileset
> you are using.
>
> *Order of Layers*
>
> Set up the layers in Tiled to this order, it helps with setting up the
> scenery layers and it seems to have some effect on layer order in the final
> binary file which is important:
>
> Collision –sets the boundaries for character movement.
> Foreground – where the characters interact. Should match the layout of the
> collision layer.
> Midground – clouds, distant landscape, etc.
> Object – set the spawn targets where objects and characters will appear.
> Hotspot – active areas to affect the characters.
>
> *Background*
>
> The Background is specified in Tiled by setting a map variable called
> backgroundIndex.
>
> *Save your map*
>
>  File; Save As… Tiled Map Files. This is the format that you can come back
> and edit later.
>
> *Export as a Replica Island binary*
>
> File ; Export As… and choose Replica Island Map File
>
> *To test your created level*
>
> First make a copy of the first level of RI.
>
> It is a .bin file with this filename: \res\raw\level_0_1_sewer.bin
>
> You may have to make a copy outside of the folder it’s in otherwise Eclipse
> will give warning messages because it tries to interpret the file.
>
> Then overwrite level_0_1_sewer.bin with the exported bin file from the Tiled
> application.
>
> Rebuild in Eclipse and Run it on your android phone.
>
> *Thank you*
>
> Thank you Chris for making your source code available, very generous of you.
>
> Thank you to all the folks who posted information on level editing to the
> Replica Island discussion group, it was very helpful.
